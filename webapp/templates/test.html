Guardrail Evaluation

Tests the embedding-similarity policy hook itself (the same logic as find_policy_conflicts / find_policy_conflicts_model_response in tourist_assistant_hooks.py) directly against a labeled dataset — no live agent or LLM call involved.

Hook type

Dataset
{{ dataset_name }} (split: {{ dataset_split }})

Metrics

Prompt-level moderation

Embeds each unsafe prompt and checks it against the policies before generation would happen.

  • block_fired_rate — a policy block fired at all (any hit above threshold), over prompts that are ground-truth unsafe.
  • any_hit_match_rate — at least one fired hit's policy matches the category(ies) the prompt is actually flagged unsafe for.
  • top_hit_match_rate — the single closest hit's policy matches one of those categories.

QA-moderation (response)

Embeds each response (both safe and unsafe) and checks whether what was actually said is unsafe.

  • precision / recall — a block should fire only on unsafe responses; precision = of responses flagged, how many were truly unsafe; recall = of truly unsafe responses, how many got flagged.
  • safety_accuracy — unsafe responses flagged unsafe and safe responses left unflagged, over all rows.
  • any_hit_match_rate / top_hit_match_rate — same definitions as above, computed only over the actually-unsafe responses.
