Response enforcement

How to handle a response that a policy check flags. "Base model" runs with no policy enforcement at all. "Base model + policies appended to prompt" runs the same unenforced model but with the alcohol_off_topic, cybersecurity_risks, financial_crimes_and_illegal_trading, and private_information policies added to its system prompt, so compliance relies on the model following instructions rather than any hook. "Hard block" refuses a flagged response outright. The critique & revise modes instead ask a model to reflect on the response against the policy and rewrite it — see critique_enforcement.py. The last option runs all five on the same prompt for comparison.
